# Contributing to Keploy Blog Website

Thank you for your interest in contributing to the Keploy Blog Website! We welcome contributions from everyone.

## Code of Conduct

Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms.

## How to Contribute

1.  **Fork the repository**: Click the "Fork" button at the top right of this page.
2.  **Clone your fork**:
    ```bash
    git clone https://github.com/YOUR_USERNAME/keploy-blog-website.git
    cd keploy-blog-website
    ```
3.  **Create a branch**:
    ```bash
    git checkout -b feature/my-new-feature
    ```
4.  **Make your changes**: Write your code and improve the project!
5.  **Test your changes**: Ensure everything works as expected.
    ```bash
    npm run dev
    # If tests are available
    npm test
    ```
6.  **Commit your changes**:
    ```bash
    git commit -m "feat: Add some amazing feature"
    ```
7.  **Push to your fork**:
    ```bash
    git push origin feature/my-new-feature
    ```
8.  **Submit a Pull Request**: Go to the original repository and click "New Pull Request".

## Style Guide

-   Use appropriate semantic HTML.
-   Follow the existing code style (Prettier/ESLint).
-   Use meaningful variable and function names.
-   Write clear and concise commit messages.

## Reporting Bugs

If you find a bug, please create an issue describing:
1.  What you expected to happen.
2.  What actually happened.
3.  Steps to reproduce the issue.

Happy contributing!
